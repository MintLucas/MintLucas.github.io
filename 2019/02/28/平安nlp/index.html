Posted on 2019-02-28 | In hide

首先必然是自我介绍：

面试官你好，我的名字是李志鹏，大家都喊我志鹏，来自河南洛阳，现在就读于科大苏研院，方向是大数据与人工智能，自己也很喜欢这个方向，所以以后想从事cv或nlp相关工作，现在是研一在读，上半学期主要从事图像算法的学习研究，对nlp停留在概念层面，没有动手跑过算法，下半学期打算做一个nlp的项目，最近开始准备暑期实习，在看剑指offer和c++书籍，我平时喜欢跑步，天气好的情况下一周会跑20公里左右。

谈一谈你对未来的规划

想先寻找一个工程性岗位，熟悉整个cv或Nlp是如何部署到商用场景及发挥价值，然后再考虑从事算法的研究岗位。

语言方面：

  • C++：指针和引用的区别，引用相当于取别名，更改即更改原变量，指针是地址；
    引用相比指针具有：非空性；不可更改性；不用检查；
  • python：
    • 常用的数据结构dict,set,map,tuple和list的区别，tuple不可更改list可更改；
    • 如何遍历一个dict，用迭代器，细节忘记了；

算法：

  1. 让你选list或者队列实现一个栈，只有入栈出栈操作，你会选哪个，时间复杂度是多少，如果用链表时间复杂度是多少，回答O(n)，看来是错的；

    解决方法：使用两个栈_data和_min，_min做为辅助栈（每次压入_min的都是_data的最小值）。元素value入栈时，将value和_min栈顶元素做比较，如果value小于等于_min.top（），将value分别push到_data和_min，否则vlaue只push到_data；元素出栈时，将_data栈顶元素和_min栈顶元素做比较，如果_data.top（）等于_min.top（），_data和_min都执行pop操作，否则只执行_data.pop（）,时间复杂度均为O(1) 。

  2. 快排和冒泡，有啥区别怎么实现的—很基础，就不记录了

  3. 介绍一下动态规划，分析问题自顶向下，求解自底向上，比如斐波那契数列，自顶向下求解会有很多重复操作，自底向下动规f2 = f1 + f0；
  4. 深度学习了解多少—花书看了第一章和cnn，rnn两章-，cnn和rnn我们是怎么优化的，就是loss是什么怎么优化—用交叉墒作为loss,然后采用反向传播算法，就是根据loss反向计算梯度。
  5. 机器学习决策树很重要，提升方法，随机森林知道吗，讲一下—boosting是多个弱分类器组合成更好的分类器，随机采样，拿了再放回，随机森林基于决策树的一种优化方法，太久了不记得了。

你有什么想问的，我们有北京深圳和上海，你会选哪个？

上海，离苏州比较近，我了解到贵司nlp是金融知识图谱，面试官笑着说知识图谱那个还不叫智能，现阶段都是手动建的，现在主要做教育阅卷等，比如讯飞那个高考机器人，科普了一些应用等没仔细听在想问题；如果我下半学期想做个项目有什么推荐的吗，—学校没有那个环境，也没设备，还是早点来公司有氛围。后续会给你个网址很简单的一些情商测试你填一下就好，应该在下周了。

太紧张了就问了这两个问题，面试官人还是非常好的，很多笑容很和蔼，让我也轻松了很多，愉快的面试。
